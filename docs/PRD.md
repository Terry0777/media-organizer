## 1. 产品概述

### 1.1 产品定位
一款纯本地的媒体文件管理工具，通过用户自定义标签 + 强大的元数据筛选，帮助用户高效整理、浏览和检索大量图片/视频文件。**无需联网，纯前端 + Rust 后端，隐私安全。**

### 1.2 目标用户
- 摄影师/视频创作者（大量素材管理）
- 普通用户（手机/相机照片积累过多）
- 设计/媒体从业者（项目素材整理）
- **隐私敏感用户（不愿上传云端）**

### 1.3 核心价值
| 痛点 | 解决方案 |
|------|----------|
| 文件多了找不到 | 元数据筛选 + 标签管理 |
| 整理耗时耗力 | 批量操作 + 智能分组 |
| 浏览体验差 | 现代化画廊 + 多视图切换 |
| 隐私担忧 | **纯本地运行，无需联网** |
| 性能问题 | **Rust 后端，高性能处理** |

### 3.2 技术选型

| 模块 | 技术选型 | 理由 |
|------|----------|------|
| 前端框架 | React + TypeScript | 生态成熟，组件丰富 |
| 桌面容器 | **Tauri v2** | Rust 后端，单二进制，体积小，性能好 |
| 后端逻辑 | **Rust (Tauri Commands)** | 高性能，内存安全，无 Python 依赖 |
| 数据库 | **SQLite + rusqlite** | 轻量，Rust 原生支持，无需联网 |
| 缩略图生成 | **image + ffmpeg** (Rust crate) | Rust 原生，高性能 |
| 文件监控 | **notify** (Rust crate) | 跨平台文件系统事件 |
| 元数据提取 | **kamadak-exif + meta** | Rust 原生 EXIF 解析 |

### 3.3 数据模型

```sql
-- 媒体文件表
CREATE TABLE media_files (
    id INTEGER PRIMARY KEY,
    file_path TEXT UNIQUE NOT NULL,
    file_type TEXT NOT NULL,  -- image/video
    file_size INTEGER,
    width INTEGER,
    height INTEGER,
    duration INTEGER,  -- 视频时长 (秒)
    created_at TIMESTAMP,
    modified_at TIMESTAMP,
    taken_at TIMESTAMP,  -- 拍摄时间
    device TEXT,
    gps_lat REAL,
    gps_lon REAL,
    checksum TEXT,  -- 用于去重
    thumbnail_path TEXT
);

-- 标签表（纯手动，无 AI）
CREATE TABLE tags (
    id INTEGER PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    parent_id INTEGER REFERENCES tags(id),
    color TEXT,  -- 标签颜色（可选）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 媒体 - 标签关联表
CREATE TABLE media_tags (
    media_id INTEGER REFERENCES media_files(id),
    tag_id INTEGER REFERENCES tags(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (media_id, tag_id)
);

-- 虚拟相册表
CREATE TABLE albums (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    cover_media_id INTEGER REFERENCES media_files(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 相册 - 媒体关联表（软链接）
CREATE TABLE album_media (
    album_id INTEGER REFERENCES albums(id),
    media_id INTEGER REFERENCES media_files(id),
    position INTEGER,
    PRIMARY KEY (album_id, media_id)
);

-- 全文搜索索引
CREATE VIRTUAL TABLE media_search USING fts5(
    file_path,
    content='media_files',
    content_rowid='id'
);
```

### 4.2 核心交互流程

#### 扫描流程
```
用户选择文件夹 → 开始扫描 → 显示进度 → AI 分析 → 生成标签 → 完成
```

```
选中媒体 → 右键/快捷键 → 添加标签 → 选择/输入标签 → 确认
```

```
输入搜索词 → 实时建议 → 执行搜索 → 展示结果 → 可进一步筛选
```

## 5. 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首次扫描速度 | 100 文件/秒 | 仅索引，不含 AI 分析 |
| AI 标签速度 | 10 文件/秒 | 单张 GPU 推理 |
| 搜索响应时间 | < 200ms | 10 万文件规模 |
| 缩略图加载 | < 50ms | 从缓存读取 |
| 内存占用 | < 500MB | 空闲状态 |
| 启动时间 | < 3s | 冷启动 |

## 6. 隐私与安全

### 6.1 隐私保护
- ✅ 所有数据本地存储，不上传云端
- ✅ AI 模型本地运行，无需联网
- ✅ 可选加密数据库（敏感标签）
- ✅ 人脸功能默认关闭，需用户主动开启

### 6.2 数据安全
- ✅ 文件操作采用软链接，原文件不移动
- ✅ 删除操作进入回收站，可恢复
- ✅ 定期自动备份数据库
- ✅ 操作日志记录（可审计）
